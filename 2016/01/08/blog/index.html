MAC系统介绍

Mac系统是基于BSD类型操作系统完善修改的,和Linux有着很近的血缘关系,所以很多的Linux指令在MacOS的终端中都是有效且快捷的.下面给大家介绍一些基本的终端指令.这是作为编程开发者的基本必修课.

独立博客

很多人已经养成了坚持写博客的习惯，但是论坛博客总有兴盛和衰落，而且最悲剧的是，通常不会持续几年，而我们做一件事是会做很久的，所以最后的结果就是不停地搬博客，博主有过这样的惨痛的经历，深知其中的苦与泪。于是一些优秀的程序猿开始找寻简便快捷长久而不失水准的发布博客的方法，于是便有了gitPage+Hexo的发布博客的方式。

Gitpage的创建

gitPage是github推出一个可以显示静态网页的服务。而对于我们的博客来说，基本除了评论的模块之外，剩下的都是静态网页，所以完全可以满足我们的需求，我们将我们的博客做成静态网页传到gitPage上就行，然后我们就可以在浏览器里访问到我们的网站了。

  1. User/Organization Pages 个人或公司站点

    1) 使用自己的用户名，每个用户名下面只能建立一个；

    2) 资源命名必须符合这样的规则username/username.github.com；

    3) 主干上内容被用来构建和发布页面

  2. Project Pages 项目站点

    1) gh-pages分支用于构建和发布；

    2) 如果user/org pages使用了独立域名，那么托管在账户下的所有project pages将使用相同的域名进行重定向，除非project pages使用了自己的独立域名；

搭建本地环境hexo

1 在安装hexo之前先安装Nodejs（直接去官网也好，百度也可，下载一个安装包，安装就行，简便快捷，这个方式比较推荐。）
2 安装完hexo之后打开终端输入命令

sudo npm install -g hexo

如果不出意外，本地hexo环境搭建完成。

3 在本地生成的hexo文件下有一个source文件夹打开有一个以.md为后缀的文件，以后我们写博客就要在这个文件里面写。如果要博客分条就重新在创建一个.md文件。
4 当在.md里面写完博客的时候你可以在本地查看你在网页上博客的显示样式
1）打开终端 （此命令含义 让hexo生成的网页在本地浏览器上可见，本人理解）

sudo hexo s

2）在本地浏览器输入以下命令查看写的博客。 

localhost:4000

3）在终端输入

sudo hexo g

把写好的网页文件保存在hexo文件夹下，打开Finder在公共（或者public）里面会发现多出好多文件以及文件夹，就是hexo生成的，（记住此位置，以后会用到）；

Git环境搭建

下载GitHub Desktop客户端安装本地。
Git环境搭建 git的搭建，首先需要申请一个github账号。 有了账号之后，登陆进去，new一个repository,仓库的名字必须是[用户名.github.io],注意,这里的名字是固定的,主要作用是限制每个账号只能有一个GitPage主页. 然后，下载一个github客户端，用于将我们的静态网页上传到刚才创建的代码仓库中。 这里解释一下，github上传代码和文件有两种，条条大路通罗马，不论用何种办法，能把东西传到github上的方法就是好方法。这里我采用的是客户端上传的方法，我个人觉得这样在使用的过程中清晰明了，操作快捷一些。部分网友喜欢使用命令行的方式操作git,我也是比较支持的,看个人爱好吧.
安装完成github Desktop之后，登录你的github账号，点击左上角的（+）加号，把你的仓库clone到本地。
